01 · Web Application

Community Communication System

A web application designed for a real client as part of a university web development module. The client wanted a platform that could be used by local towns to promote small businesses through an interactive map-based interface.

The application included main, secondary and tertiary street navigation, allowing users to explore different areas of the town and view local organisations. Businesses could use the platform to present their details and promote themselves through content such as descriptions, links and promotions.

My main contribution focused on the visual design of the interactive map, the location box used within the map interface, and the view/edit organisation details functionality for both organiser and admin users.

Key Features

  • Interactive map with main, secondary and tertiary street navigation.
  • Organisation pages for local businesses and community groups.
  • Promotion areas where businesses could present offers and updates.
  • Role-based access for organisers and administrators.
  • View and edit organisation details functionality.
  • Location box to help users understand where they are within the map.

Technologies Used

  • Java
  • Spring Boot
  • Thymeleaf
  • MariaDB
  • HTML
  • CSS
  • JavaScript
  • JUnit

02 · DevOps

DevOps Pipeline

A CI/CD deployment pipeline developed as part of a university DevOps module to automate the build, testing and deployment workflow of the “Edit Organisation Details” functionality within a Spring Boot web application.

The pipeline focused on ensuring that organiser and admin users could reliably update organisation information through a fully automated workflow that included testing, artifact generation, containerisation and deployment.

Key Features

  • Automated build and deployment workflow using Jenkins.
  • Testing of the Edit Organisation Details functionality.
  • Containerised application deployment using Docker.
  • Infrastructure provisioning with Terraform on Azure DevTest Labs.
  • Artifact storage and retrieval through GitLab Packages.
  • Nginx reverse proxy and SSL configuration.
  • Selenium and JMeter testing integration.

Technologies Used

  • Jenkins
  • Docker
  • Terraform
  • GitLab
  • Nginx
  • Azure
  • Selenium
  • JMeter

03 · UX Prototype

SignQuest Prototype

A university UX design project focused on creating an accessible learning experience for children with hearing disabilities through gamified British Sign Language and lip-reading activities.

The prototype was designed in Figma and aimed to make learning more engaging through storyline progression, rewards, interactive tasks and child-friendly visual design. Accessibility and usability were central to the project, with particular focus on visual communication, simplicity and cognitive load reduction.

Key Features

  • Interactive BSL and lip-reading learning tasks.
  • Storyline progression system with rewards and unlockable stages.
  • Child-friendly visual interface and navigation.
  • Accessibility-focused design for hearing-impaired children.
  • Gamification elements including diamonds, stages and progression.
  • Prototype designed and tested in Figma.

Research & Evaluation

The project included user research, heuristic evaluation, WCAG accessibility considerations, questionnaires and think-aloud usability testing to evaluate engagement, usability and accessibility.
